# Cogniforge SRS 验收清单与缺口分析

日期: 2026-03-03
基线文档: `/Users/asteroida/work/Cogniforge/SRS.md`

## 结论摘要

- 按完整 SRS 口径评估，项目整体完成度约为 60%。
- 核心业务骨架已存在，但仍有多处属于“接口已存在、流程未闭环”。
- 当前更接近可演示 MVP，而不是可按 SRS 完整验收的版本。
- 项目存在若干超出原始 SRS 的扩展功能，如 `cog-test`、资源解读、快速笔记、管理后台。

## 评级标准

- 已完成: 需求已有可用实现，前后端主流程基本闭环。
- 部分完成: 有后端能力或前端入口，但流程未闭环，或实现深度明显低于 SRS。
- 未完成: 缺少关键实现，无法支撑需求验收。
- 超出需求: 不在 SRS 主体内，但仓库已有扩展实现。

## 五、前后端闭环情况

### 已闭环

- 注册、登录、获取当前用户
- 创建问题并生成学习路径
- 创建 Model Card
- 生成反例
- 生成跨域迁移建议
- 创建练习任务和提交练习
- 认知挑战生成与回答
- 资源链接管理与 AI 解读
- `cog-test` 会话、SSE 对话、报告导出

### 半闭环

- 学习路径推进: 后端有接口，前端无推进动作
- Review 复盘: 数据 CRUD 有了，但没有自动生成和报告输出
- SRS: 后端有 schedule/review，前端缺少将 Model Card 加入计划入口
- 演化日志: 详情页有时间线，但没有版本对比和摘要
- 聊天: 功能能跑，但更像通用聊天，不是结构化“学习反馈对话”

### 未闭环

- 模型库搜索/语义检索
- RAG
- pgvector 知识迁移
- 学习路径动态调优
- 周/月自动复盘
- 学习报告导出
- 版本历史管理与可视化差异

## 六、关键缺口清单

### P0 缺口

- 数据层未按 SRS 使用 PostgreSQL/pgvector。
- 学习路径只有展示，没有前端推进闭环。
- Review 仅是手工记录，不是自动复盘模块。
- SRS 缺少“加入排程”前端入口。
- Model Card 缺少搜索和版本链。

### P1 缺口

- 聊天与问题反馈未做结构化误解分析。
- 跨域迁移未形成“建议 -> 尝试 -> 记录”的完整回路。
- 复盘报告未与 EvolutionLog、Practice、Challenge、SRS 统一。

### P2 缺口

- 无 RAG、无向量召回、无知识片段化。
- 无性能基准、无验收级自动化测试。
- 安全基线未达生产要求。

## 七、超出 SRS 的现有扩展

- 管理后台
- 多 LLM 提供商配置
- 邮件配置
- 快速笔记
- 资源链接 AI 解读
- 认知测试 `cog-test`
- Android Capacitor 壳

这些功能可以保留，但在补齐主线需求时不应继续分散主资源。

## 八、建议验收口径

后续推进应采用双口径管理:

- 口径 A: 完整 SRS 验收
- 口径 B: 可交付 MVP 验收

当前仓库已接近口径 B，但距离口径 A 仍有明显差距。
