# Cogniforge SRS 验收清单与缺口分析

日期: 2026-03-04
基线文档: `/Users/asteroida/work/Cogniforge/SRS.md`

## 结论摘要

- 按完整 SRS 口径评估，项目整体完成度约为 80%。
- 核心学习主流程已经闭环，当前已不再是“可演示 MVP”，而是可内测的 beta 状态。
- PostgreSQL/pgvector、轻量 RAG、Review、SRS、版本对比、结构化反馈、管理后台和后端 smoke tests 均已落地。
- 鉴权持久化、RAG 召回日志和前端 Dashboard smoke 也已补齐第一版。
- 剩余缺口主要集中在:
  - 架构与安全的最后一层生产化
  - RAG 可观测性的进一步产品化
  - 前端自动化测试扩面
  - 文档与验收材料收尾

## 评级标准

- 已完成: 需求已有可用实现，前后端主流程闭环，且已有基本验证。
- 部分完成: 已有实现，但深度、可观测性或稳定性仍低于 SRS 最终目标。
- 未完成: 缺少关键实现，无法支撑需求验收。
- 超出需求: 不在 SRS 主体内，但仓库已有扩展实现。

## 五、前后端闭环情况

### 已闭环

- 注册、登录、当前用户、双 token 刷新、登出
- 创建问题、学习路径推进、问题回答历史
- Model Card 创建、搜索、相似卡片、演化日志、版本 compare
- Counter examples、迁移建议
- Practice 任务 CRUD、提交反馈
- Review 自动生成、保存、编辑、导出
- SRS 加入排程、到期复习、评分更新
- Challenge 生成、回答、筛选
- Resource 管理与搜索
- `cog-test` 会话、SSE、停止诊断、报告导出
- 真实 PostgreSQL + pgvector 迁移验证
- Retrieval logs / summary API 与 Dashboard 召回观察卡
- Dashboard 浏览器 smoke 脚本

### 半闭环

- 学习路径动态调优: 有推进，但无基于反馈的自动重排
- 迁移闭环: 有建议，无迁移实践记录
- Review 与 cog-test 的更深层整合: 有聚合，但不是完整认知回顾系统
- 管理后台 LLM 能力: 已支持 OpenAI/Anthropic/Ollama/Qwen，但仍缺更丰富 provider 生态

### 未闭环

- RAG 质量评估与更完整调试面板
- 前端自动化测试扩面
- SRS 级正式性能和验收基准

## 六、关键缺口清单

### P0 缺口

- 更新文档与验收口径，使之反映当前真实完成度
- 前端 smoke 从单条 happy path 扩到更多主流程

### P1 缺口

- RAG 质量评估、命中效果分析与更完整调试面板
- 迁移尝试记录闭环

### P2 缺口

- 统一的 LLM 调用指标、耗时和失败可观测性
- 更正式的 prompt/workflow 状态编排
- 移动端产品化
