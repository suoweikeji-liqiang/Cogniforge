**Step 2 — Create `_elevate_srs_priority_if_blind_spots` helper:**

Add as a module-level async function in `cog_test.py`:

```python
async def _elevate_srs_priority_if_blind_spots(
    session: CogTestSession,
    db: AsyncSession,
) -> None:
```

Logic:
1. Check if session has any blind spots: `select(CogTestBlindSpot).where(CogTestBlindSpot.session_id == session.id).limit(1)`. If none found, return early.
2. Get `model_card_id` from session: `model_card_id = session.model_card_id`. If `None`, return early (session started without model card link).
3. Find existing `ReviewSchedule`: `select(ReviewSchedule).where(ReviewSchedule.user_id == session.user_id, ReviewSchedule.model_card_id == model_card_id)`.
4. If no schedule exists, create one: `schedule = srs_service.schedule_card(model_card_id, session.user_id)`, then `db.add(schedule)` and `await db.flush()`.
5. Apply quality=0: `srs_service.process_review(schedule, quality=0)` — this resets interval to 1 day, pushing the card to the front of the review queue.
6. Do NOT call `db.commit()` — the caller owns the transaction.

Follow the exact pattern from 04-RESEARCH.md Pattern 2.

**Step 3 — Wire the helper into the stop endpoint:**

The stop endpoint (`POST /sessions/{session_id}/stop`) should already exist or will be created by Phase 2 plans. After setting `session.status = "stopped"`, add:

```python
await _elevate_srs_priority_if_blind_spots(session, db)
```

This must be called BEFORE `await db.commit()` so the SRS schedule mutation is included in the same transaction.

**Step 4 — Wire the helper into the session_complete path:**

In the SSE stream endpoint (the one that calls `engine.run(db)`), after the EventSourceResponse generator completes naturally (session_complete), reload the session from DB and call:

```python
session = await db.get(CogTestSession, session_id)
if session and session.status == "completed":
    await _elevate_srs_priority_if_blind_spots(session, db)
    await db.commit()
```

If the stream endpoint structure makes this difficult (EventSourceResponse is a streaming response), an alternative is to add the elevation call inside the engine's `_finalize_session` or at the end of the stop endpoint only. The stop endpoint is the guaranteed trigger point — document this choice.
