# Plan 02: LLM Streaming + SSE Infrastructure

## Objective
Extend LLMService with async streaming capability and add sse-starlette dependency. Create a test SSE endpoint that proves tokens stream correctly from LLM to client.

## must_haves
- sse-starlette added to requirements.txt
- LLMService.stream_generate() async generator yields tokens
- Test SSE endpoint at /api/cog-test/stream-test returns streamed tokens
- Works with both OpenAI and Anthropic providers

## Tasks

<task id="2">
<title>Add stream_generate() to LLMService</title>
<details>
Add a new async generator method to `LLMService` in `las_backend/app/services/llm_service.py`:

```python
async def stream_generate(
    self,
    messages: list[dict],
    system_prompt: str = "",
    provider_type: str | None = None,
    model_id: str | None = None,
    temperature: float = 0.7,
) -> AsyncGenerator[str, None]:
```

Key implementation points:
- Use `openai.AsyncOpenAI` (not sync `OpenAI`) with `stream=True`
- For Anthropic: use `anthropic.AsyncAnthropic` with `stream=True` via `client.messages.stream()`
- Yield each token/chunk as a string
- Resolve provider and model using existing `_get_active_provider` and `_get_default_model`
- Add `from typing import AsyncGenerator` import
- For OpenAI: iterate `async for chunk in response:` and yield `chunk.choices[0].delta.content`
- For Anthropic: iterate events and yield text deltas
- Handle None content chunks (skip them)
</details>
<verification>
Method exists and can be imported: `from app.services.llm_service import llm_service; assert hasattr(llm_service, 'stream_generate')`
</verification>
</task>
