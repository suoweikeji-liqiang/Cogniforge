Create `las_frontend/src/stores/cogTest.ts` using the composition API pattern (same as auth.ts).

Define a `Message` interface:
```typescript
interface Message {
  role: 'guide' | 'challenger' | 'user'
  content: string
  streaming?: boolean
}
```

Define `SessionStatus` type: `'idle' | 'connecting' | 'streaming' | 'waiting' | 'stopped' | 'completed'`

Define `SessionSummary` interface for history list:
```typescript
interface SessionSummary {
  id: string
  concept: string
  status: string
  created_at: string
  score?: number | null  // optional: v1 backend does not return score in list endpoint; included for forward-compatibility when backend adds it
}
```

Store state refs:
- `sessionId: ref<string | null>(null)`
- `concept: ref<string>('')`
- `status: ref<SessionStatus>('idle')`
- `messages: ref<Message[]>([])`
- `currentRound: ref(1)`
- `maxRounds: ref(3)`
- `error: ref<string | null>(null)`
- `sessions: ref<SessionSummary[]>([])` — for history list
- `let eventSource: EventSource | null = null` (non-reactive, module-level)

Implement `getSSEBaseUrl()` helper using Capacitor pattern from api/index.ts:
```typescript
import { Capacitor } from '@capacitor/core'
const getSSEBaseUrl = () => {
  if (Capacitor.isNativePlatform()) {
    return localStorage.getItem('api_server_url') || 'http://10.0.2.2:8002/api'
  }
  return '/api'
}
```

Implement `startSession(conceptName: string)`:
1. POST `/cog-test/sessions` with `{ concept: conceptName, max_rounds: 3 }` via api
2. Set sessionId, concept, status='connecting', clear messages
3. Call `_openStream()`

Implement `_openStream()` (private, not exported):
1. Get token from `useAuthStore().token`
2. Build URL: `${getSSEBaseUrl()}/cog-test/sessions/${sessionId.value}/stream?token=${token}`
3. Create `new EventSource(url)`
4. Add named event listeners for each SSE event type:
   - `session_start`: parse JSON, set maxRounds, set status='streaming'
   - `turn_start`: parse JSON, push new Message `{ role: data.role, content: '', streaming: true }`, set currentRound
   - `token`: append `e.data` to last message's content (mutate in place — `last.content += e.data`)
   - `turn_complete`: set last message's streaming=false, set status='waiting'
   - `round_complete`: parse JSON, set currentRound
   - `session_complete`: set status='completed', close eventSource, set to null
   - `error` (named SSE event): parse JSON, set error message, set status='stopped', close eventSource
5. Set `eventSource.onerror` handler: if status is not completed/stopped, set status='stopped' and error='Connection lost', then close eventSource

Implement `submitUserTurn(text: string)`:
1. Guard: if no sessionId, return
2. Push user message: `{ role: 'user', content: text }`
3. Set status='streaming'
4. POST `/cog-test/sessions/${sessionId.value}/turns` with `{ text }`

Implement `stopSession()`:
1. Guard: if no sessionId, return
2. POST `/cog-test/sessions/${sessionId.value}/stop`
3. Close eventSource, set to null
4. Set status='stopped'

Implement `fetchSessions()`:
1. GET `/cog-test/sessions`
2. Set sessions.value = response.data

Implement `resetSession()`:
1. Close eventSource if open
2. Reset all refs to initial values (sessionId=null, concept='', status='idle', messages=[], etc.)

Export all state refs and action functions from the store.

CRITICAL: Do NOT use `eventSource.onmessage` — the backend sends named events. Use `addEventListener('event_name', handler)` for each event type.
CRITICAL: Call `eventSource.close()` in session_complete, error, onerror, and stopSession to prevent auto-reconnect.
