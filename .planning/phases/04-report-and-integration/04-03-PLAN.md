**Context:** Phase 2 Plan 02-03 will create the SSE stream endpoint (`GET /sessions/{id}/stream`). This gap closure task modifies that endpoint to call `_elevate_srs_priority_if_blind_spots` after the engine generator exhausts.

**The problem with post-EventSourceResponse DB access:** `EventSourceResponse` consumes the async generator. Once the response is returned, FastAPI's dependency lifecycle may close the `db` session. We cannot reliably run DB operations AFTER `EventSourceResponse(engine.run(db))` returns.

**The solution:** Wrap `engine.run(db)` in a new async generator that:
1. Yields all events from `engine.run(db)` (pass-through)
2. After the inner generator exhausts (all events yielded), runs the elevation logic INSIDE the generator — this keeps the DB session alive because the generator is still executing within the EventSourceResponse lifecycle
3. Cleans up the engine registry

**Implementation:**

Add a helper async generator function in `cog_test.py`:

```python
async def _stream_with_elevation(
    engine: CogTestEngine,
    session_id: str,
    db: AsyncSession,
):
    """Wrap engine.run() to trigger SRS elevation after natural session completion."""
    try:
        async for event in engine.run(db):
            yield event
    finally:
        # Post-stream: check if session completed naturally and elevate SRS
        try:
            session = await db.get(CogTestSession, session_id)
            if session and session.status == "completed":
                await _elevate_srs_priority_if_blind_spots(session, db)
                await db.commit()
        except Exception:
            pass  # best-effort — stop endpoint is the guaranteed fallback
        unregister_engine(session_id)
```

Then in the stream endpoint (which Phase 2 Plan 02-03 creates), replace:
```python
return EventSourceResponse(engine.run(db))
```
with:
```python
return EventSourceResponse(_stream_with_elevation(engine, session_id, db))
```
